# 何时使用
- 用户明确说"总结飞书邮件""帮我整理邮件""生成邮件日报/周报""把今天/本周的邮件总结一下"。
- 用户希望快速浏览未读邮件并提炼待办事项时使用。
- 仅在用户表达"总结邮件"意图时才加载 — 避免误触发。

# 执行步骤
1. 调用 lark-mail Skill 拉取指定时间范围的邮件列表（默认最近 24 小时；若用户指定"本周""今天"则按其范围）。
2. 按发件人、主题、紧急程度对邮件分类：
   - 待回复邮件（含明确问询或请求）
   - 通知/公告类邮件
   - 抄送知会类邮件
3. 提炼每封重要邮件的核心内容（1-2 句话）+ 是否需要行动。
4. 读取 references/summary-template.md 套用总结模板。
5. 生成结构化总结报告，按"待办优先级"排序输出。
6. 询问用户是否需要将总结回发到自己邮箱存档（可选）。

# 输出格式
```text
📬 飞书邮件总结 · <日期范围>
共 <N> 封邮件（未读 <M> 封）

🔴 待回复（高优先级）：
· [发件人] <主题>
  摘要：...
  建议动作：<回复要点> — 截止 <日期>

🟡 待关注：
· [发件人] <主题> — <一句话摘要>

🟢 知会类（无需回复）：
· <发件人> · <主题>
· <发件人> · <主题>

📌 建议今日处理：<N> 项
```

# 错误处理
- 拉取邮件失败：提示用户检查飞书邮箱授权状态，或提供时间范围重试。
- 邮件数量为 0：告知用户该时间段内无新邮件，询问是否扩大时间范围。
- 邮件内容过长无法全部提炼：优先处理标记为"重要"或"未读"的邮件，并告知用户已截断。
- 用户对总结颗粒度不满意：支持"再详细一些""只看待回复的"等二次细化指令。

# 注意事项
- 涉及敏感内容（薪资、合同、人事）的邮件，摘要中只提示"含敏感信息，请直接查看原文"，不展开内容。
- 不自动代替用户回复邮件，仅生成总结与建议。
